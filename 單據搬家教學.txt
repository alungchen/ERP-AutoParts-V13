------------------------------------------------------
📜 歷史單據搬家教學指南 (全自動流程)
------------------------------------------------------
本教學將引導您如何將舊系統的歷史單據（銷貨單、報價單、進貨單等）
「整批、自動化」搬移到新的 Cloudflare D1 資料庫中。

------------------------------------------------------
👉 第一步：執行自動化爬蟲 (抓取 CSV)
------------------------------------------------------
請打開 VS Code 終端機 (Terminal)，根據您要抓取的「單據類型」與「日期範圍」
輸入以下指令（請依需求修改參數）：

指令格式：
node scripts/scrape_documents.cjs --type=<類型> --start=<開始日期> --end=<結束日期>

範例 (抓取 2024 全年的銷貨單)：
node scripts/scrape_documents.cjs --type=sales --start=2024-01-01 --end=2024-12-31

說明：
- type 支援：sales (銷貨), quotation (報價), inquiry (詢價), purchase (進貨)
- 程式會自動打開瀏覽器，請在「紅框提示」出現時手動登入。
- 登入後程式會自動按日期倒著抓取，抓完後會存在 output/ 資料夾。

------------------------------------------------------
👉 第二步：轉換 CSV 為 SQL 指令
------------------------------------------------------
抓取完成後，請執行以下指令將 CSV 檔案轉換成資料庫看得懂的 SQL 檔案：

node scripts/generate_document_sql.cjs

(說明：這會產生 output/import_documents.sql 檔案，準備匯入。)

------------------------------------------------------
👉 第三步：匯入雲端資料庫 (正式上線)
------------------------------------------------------
最後，輸入以下指令將資料同步到雲端：

npm run db:import-docs:remote

(說明：這會將產生的單據資料正式寫入 Cloudflare D1 雲端資料庫。)

------------------------------------------------------
💡 小技巧：
------------------------------------------------------
1. 建議分批處理：例如一次抓半年或一年，避免單次爬取時間過長導致超時。
2. 資料完整性：如果單據裡的零件號碼在「產品資料庫」中已存在，系統會自動關聯並顯示正確的品名與規格。
3. 如果只是想在「本機測試」，第三步可以改跑：npm run db:import-docs:local
